This section of the super computing tutorial series is going to cover how to use MPI with MPI4py and the Python programming language.

If you're just joining us now, you should already have either a super-computer (we made one out of Raspberry Pis), or you can actually use MPI just for parallel computing with multiple cores in a CPU.

MPI is short for Message Passing Interface. The idea is to have an interface to, you guessed it, pass messages. The idea is to share variables and other data between nodes or cpus. The idea behind that is that you can have the calculations and processing node-specific, but you can either share the data/variables across all nodes, or send to a master node. This takes a lot of processing and time and spreads it around the network. The act of message passing is very cost-minimal.

To start, you will need Python, as well as MPI4Py.
